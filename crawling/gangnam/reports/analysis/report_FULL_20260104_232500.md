# 강남언니 병원 데이터 전체 수집 리포트

## 1. 수집 개요
- **수집 대상**: 강남언니 병원 정보 전체 (https://www.gangnamunni.com/hospitals)
- **수집 목적**: 전체 병원 데이터(약 1400개 예상) 확보
- **수집 일시**: 2026년 1월 4일

## 2. 변경된 수집 전략
기존 1회 요청 방식에서 발생한 데이터 개수 제한(최대 200개) 문제를 해결하기 위해 다음과 같이 수집 로직을 개선하였습니다.

1.  **지역 코드 분할 요청**:
    - 전체 `districtCodeList`를 하나의 요청에 보내지 않고, 115개의 개별 지역 코드로 분리하였습니다.
    - 각 지역 코드별로 API를 순차 호출하여 해당 지역의 병원 리스트를 모두 가져왔습니다.
2.  **중복 제거**:
    - 여러 지역에 겹쳐서 노출되는 병원이 있을 수 있으므로, 병원 고유 `id`를 기준으로 중복을 실시간 제거하였습니다.
3.  **안정성 확보**:
    - 다수의 HTTP 요청(수백 회 이상)이 발생하므로, 요청 간 딜레이를 주어 서버 부하를 줄이고 차단을 방지하였습니다.

## 3. 수집 결과
- **저장 파일**: `gangnam_hospitals_FULL_20260104_231939.csv`
- **최종 수집 건수**: **1,103건**
    - *참고*: 사용자가 예상한 1,400개보다 적은 이유는 지역 간 중복 제거, 폐업 병원 제외, 또는 API 검색 로직의 특성(광고 미진행 병원 노출 여부 등)일 수 있습니다.
- **수집 항목**:
    - `id`: 병원 고유 ID (추가됨)
    - `hospital_name`: 병원 이름
    - `rating`: 평점
    - `review_count`: 리뷰 수
    - `event_count`: 이벤트 수
    - `district_code`: 수집된 지역 코드

## 4. 결론
개별 지역 코드 순회 방식을 통해 API의 검색 결과 수 제한을 우회하였으며, 가능한 모든 병원 데이터를 수집하여 CSV로 저장하였습니다.
