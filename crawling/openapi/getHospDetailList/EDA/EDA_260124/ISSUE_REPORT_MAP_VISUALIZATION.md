# Issue Report: 입지 분석 시각화 지도 미표출 리포트

## 1. 개요
*   **발생 현상**: 가이드라인상 입지 분석 시 **지도(Map) 시각화**가 요구되었으나, 실제 결과물은 **수평 막대 그래프(Horizontal Bar Chart)**로 출력됨.
*   **발생 일시**: 2026-01-24 17:30
*   **분석 대상**: `피부과_병원정보_20260124_153603.csv`

## 2. 원인 분석 (Root Cause Analysis)
데이터 컬럼 전체 검수 결과, 지도 시각화에 필수적인 **좌표 데이터(위도/경도 또는 X/Y 좌표)**가 원본 데이터에 **부재**합니다.

*   **보유 컬럼**: `원본_주소`, `eqp_addr` (텍스트 형태의 주소 정보만 존재)
*   **누락 컬럼**: `lat`, `lon`, `x`, `y` 등 좌표 정보 없음
*   **결과**: 좌표값 없이 지도 위에 점(Point)을 찍을 수 없어, 차선책(Fallback)으로 분포 순위를 보여주는 막대/트리맵 그래프가 적용되었습니다.

## 3. 해결 방안 (Solution)

### 3.1 [기술적 해결] 지오코딩(Geocoding) 적용
텍스트 형태의 주소를 좌표로 변환하는 **지오코딩** 전처리가 선행되어야 합니다.

1.  **지오코딩 API 활용**: `eqp_addr`(주소) 컬럼을 API에 전송하여 위도(Lat), 경도(Lon) 컬럼 생성.
    *   *활용 가능 서비스*: VWorld(무료/공공), Kakao Map API, Google Maps API 등.
2.  **데이터 병합**: 생성된 좌표 데이터를 원본 CSV에 병합(`merge`).
3.  **시각화 툴 적용**: `folium` 또는 `plotly` 라이브러리를 사용하여 지도 시각화 구현.

### 3.2 [실행 계획 제안]
지금 즉시 지도를 생성하려면 다음 과정이 필요합니다.

1.  **Geocoding 스크립트 작성**: 주소 리스트 추출 및 변환.
2.  **좌표 데이터 확보**: (API 키 필요 여부에 따라 시간 소요 가능).
3.  **EDA 재실행**: 좌표가 포함된 데이터로 입지 분석 섹션 업데이트.

## 4. 결론
지도 시각화가 누락된 것은 **스크립트 오류가 아닌 원본 데이터의 좌표 정보 부재** 때문입니다. 향후 분석 시 **'주소 좌표 변환(Geocoding)'** 단계를 분석 파이프라인에 추가할 것을 권장합니다.
