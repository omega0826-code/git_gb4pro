## 1. 개요
사용자가 문의한 두 CSV 파일(`병원전체정보` vs `서울_강남구_피부과`) 간의 컬럼 수 차이에 대한 원인을 분석하였습니다. 분석 결과, 이는 **'데이터 수집 단계의 원본 데이터'와 '분석/활용 목적의 2차 가공 데이터' 간의 구조적 차이**로 확인되었습니다.

### 2.2. 주요 차이점 상세

#### A. 원본 파일 (`병원전체정보_...`)
*   **성격**: 건강보험심사평가원(HIRA)의 API 11종(기본, 세부, 진료과목, 교통 등)을 모두 호출하여 병합한 **Raw Data Dictionary** 형태입니다.
*   **컬럼 구성**: API에서 제공하는 원천 필드명을 그대로 사용 (`eqp_`, `dtl_`, `dgsbjt_` 등 접두어 사용).
*   **포함 정보**: 병원의 모든 세부 정보(상세 진료시간, 특수진료 여부, 교통편, 의료장비 상세 등)를 망라.

#### B. 추출 파일 (`서울_강남구_피부과_...`)
*   **성격**: 원본 데이터에서 특정 조건(서울 강남구, 피부과)으로 필터링하고, **분석에 필요한 핵심 정보만 선별(Selection)** 하여 재가공한 파일입니다.
*   **변환 내용**:
    1.  **컬럼 선택 (Selection)**: 분석에 불필요한 세부 정보(상세 코드, API 관리용 필드 등)를 제거하고 `병원명`, `주소`, `전화번호` 등 핵심 컬럼만 유지.
    2.  **컬럼명 변경 (Renaming)**: `eqp_yadmNm` → `병원명`, `eqp_addr` → `주소` 등 직관적인 한글명으로 매핑.
    3.  **파생 변수 추가 (Feature Engineering)**: 원본에는 없는 **`경도`, `위도`** 좌표 데이터가 추가됨 (주소를 기반으로 별도 지오코딩 작업 수행 추정).

## 4. 결론

두 파일의 컬럼 수 차이는 **데이터 오류가 아니라 의도된 가공 과정의 결과**입니다.

1.  **원본 파일**은 "데이터의 보존 및 망라성"을 목적으로 모든 API 정보를 담고 있어 컬럼이 많습니다.
2.  **추출 파일**은 "특정 지역/진료과 분석(EDA)"을 목적으로 **핵심 정보 선별(Selection)** 및 **사용자 친화적 가공(Renaming)**, 그리고 **위치 정보 부가(Geocoding)** 가 이루어진 결과물입니다.

따라서, 상세한 API 정보가 필요하다면 '원본 파일'을, 위치 기반 분석이나 빠른 현황 파악이 필요하다면 '추출 파일'을 사용하는 것이 적절합니다.
