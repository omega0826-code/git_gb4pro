## 🤔 어떻게 사용하나요?

MCP 서버는 **직접 실행하는 프로그램이 아닙니다.**  
AI 클라이언트(Gemini CLI, VS Code, Claude Desktop 등)가 **자동으로 호출**합니다.

### 사용 흐름

```
나(사람) → AI에게 자연어로 요청 → AI가 MCP 도구 호출 → 결과 반환
```

예시:
1. 나: *"보고서.hwpx 파일의 내용을 보여줘"*
2. AI가 내부적으로 `get_document_text` 도구를 호출
3. AI: *"보고서.hwpx의 내용은 다음과 같습니다: ..."*

> 💡 **핵심**: 도구 이름을 외울 필요 없습니다. **자연어로 요청**하면 AI가 알아서 적절한 도구를 선택합니다.

## 📂 파일 준비

hwpx-mcp-server는 `.hwpx` 파일만 처리합니다.

### .hwpx 파일이란?

- 한컴오피스 한글 2014 이후 버전에서 지원하는 **공개 문서 포맷**
- 내부적으로 XML 파일들을 ZIP으로 묶은 구조
- 한글 프로그램에서 "다른 이름으로 저장" → "HWPX" 선택하면 생성 가능

### 작업 폴더에 파일 준비

AI에게 파일 경로를 알려주면 됩니다:

```
"D:\문서\보고서.hwpx 파일의 내용을 읽어줘"
```

또는 현재 작업 폴더 안에 있으면:

```
"보고서.hwpx 파일을 열어서 내용을 보여줘"
```

## 📖 기본 사용법: 문서 읽기

**AI가 알려주는 정보:**
- 문서 제목, 저자, 작성일

## 🔬 고급 기능 (선택사항)

> 이 기능은 환경 변수 `HWPX_MCP_ADVANCED=1`로 설정해야 활성화됩니다.  
> 문서의 내부 XML 구조를 직접 다루는 전문가용 기능입니다.

### 문서 내부 구조 탐색

```
요청: "보고서.hwpx의 내부 파트(파일) 목록을 보여줘"
```

```
요청: "보고서.hwpx의 content.hpf XML 내용을 보여줘"
```

### 안전한 편집 (계획 → 미리보기 → 적용)

1단계 - 편집 계획:
```
요청: "보고서.hwpx에서 모든 '제목' 태그의 글꼴 크기를 변경하는 계획을 세워줘"
```

2단계 - 미리보기:
```
요청: "변경 사항을 미리보기로 보여줘"
```

3단계 - 적용:
```
요청: "미리보기 내용이 좋으니 적용해줘"
```

### 문서 검증

```

```


## ⚠️ 주의사항

1. **`.hwpx`만 지원** — `.hwp`, `.docx`, `.pdf` 등은 이 서버로 처리 불가
2. **파일 경로에 한글 사용 가능** — 단, 일부 환경에서 인코딩 문제 발생 가능
3. **자동 백업 활용** — 중요한 문서 편집 전 `HWPX_MCP_AUTOBACKUP=1` 확인
4. **큰 문서 주의** — `HWPX_MCP_MAX_CHARS` 값보다 큰 문서는 내용이 잘릴 수 있음
5. **동시 편집 불가** — 한글 프로그램에서 열고 있는 파일을 동시에 편집하면 충돌 가능
