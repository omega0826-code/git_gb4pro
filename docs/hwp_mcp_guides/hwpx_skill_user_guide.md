# HWPX 스킬 사용자 가이드라인

> **한글(Hancom Office)의 HWPX 문서**를 생성·편집·읽기하기 위한 실용 가이드

---

## 1. 개요

HWPX 스킬은 한글 문서(.hwpx)를 **XML 직접 작성** 방식으로 다루는 도구입니다.  
HWPX는 ZIP 기반 XML 컨테이너(OWPML 표준)로, 이 스킬을 통해 세밀한 서식 제어가 가능합니다.

> [!IMPORTANT]
> **`.hwp`(구 바이너리 형식)는 지원하지 않습니다.**  
> `.hwp` 파일은 한글에서 `파일 → 다른 이름으로 저장 → 파일 형식: HWPX`로 변환 후 사용하세요.

## 4. 사용 가능한 템플릿

### 4.1 base (기본)
- 빈 문서의 기본 골격
- 함초롬바탕 10pt, 양쪽 정렬, 160% 줄간격

### 4.2 gonmun (공문)
- 기관 공문서 양식
- 기관명(22pt 볼드), 서명자(16pt 볼드), 연락처(8pt) 스타일 포함
- 표 헤더용 회색 배경(#D6DCE4) 스타일

### 4.3 report (보고서)
- 비즈니스 보고서 양식
- 제목(20pt), 소제목(14pt), 섹션 헤더(12pt 돋움) 등 다단계 스타일
- 들여쓰기 3단계 (□, ①②③, -) 자동 지원
- 섹션 헤더에 상하단 구분선 자동 표시

### 4.4 minutes (회의록)
- 회의록 전용 양식
- 제목(18pt 볼드), 섹션 라벨(12pt 볼드)
- 연한 녹색 배경(#E2EFDA)의 표 헤더

### 4.5 proposal (제안서/사업개요)
- 시각적으로 구분된 공식 문서
- 올리브녹색(#7B8B3D) 대항목 번호 배지
- 파란색(#4472C4) 소항목 번호 배지
- 색상 배경 헤더바 + 번호 배지 시스템

## 5. 핵심 워크플로우

### 5.1 새 문서 만들기 (가장 많이 사용)

```
사용자 요청 → 템플릿 선택 → section0.xml 작성 → build_hwpx.py로 빌드 → validate.py로 검증
```

**요청 방법:**
```
"보고서 템플릿으로 [내용]을 작성한 HWPX 문서를 만들어줘"
```

### 5.2 기존 문서 편집

```
HWPX 파일 제공 → unpack.py로 풀기 → XML 편집 → pack.py로 재패키징 → 검증
```

**요청 방법:**
```
"이 HWPX 파일에서 [특정 부분]을 [수정 내용]으로 변경해줘"
```

### 5.3 텍스트 추출 / 읽기

```
HWPX 파일 제공 → text_extract.py 실행
```

**요청 방법:**
```
"이 한글 파일의 내용을 읽어줘"
"이 HWPX 파일의 표 데이터를 마크다운으로 추출해줘"
```

### 5.4 레퍼런스 기반 문서 생성 (고급)

기존 HWPX 파일의 레이아웃을 분석하여 동일한 양식으로 새 문서를 생성합니다.

```
레퍼런스 HWPX 제공 → analyze_template.py로 분석 → 스타일 추출 → 새 내용으로 빌드
```

**요청 방법:**
```
"이 양식(reference.hwpx)과 동일한 레이아웃으로 [새 내용]을 작성해줘"
```

## 9. 자주 묻는 질문 (FAQ)

### Q. `.hwp`와 `.hwpx`의 차이는?
**A.** `.hwp`는 한글의 구(舊) 바이너리 형식이고, `.hwpx`는 ZIP 기반 XML 형식(OWPML 표준)입니다. 이 스킬은 `.hwpx`만 지원합니다.

### Q. 생성된 문서를 한컴오피스에서 열 수 있나요?
**A.** 네, 한컴오피스 한글(Hangul)에서 정상적으로 열리며 편집도 가능합니다.

### Q. 표(Table)도 만들 수 있나요?
**A.** 네, 행수/열수/열너비를 지정한 표를 포함한 문서를 생성할 수 있습니다. 셀 병합(rowSpan/colSpan)도 지원합니다.

### Q. 기존 양식을 그대로 활용할 수 있나요?
**A.** 네, 레퍼런스 HWPX 파일을 제공하면 `analyze_template.py`로 구조를 분석하여 동일한 레이아웃으로 새 문서를 생성할 수 있습니다.

### Q. 이미지 삽입이 가능한가요?
**A.** 현재 스킬에서는 텍스트와 표 중심의 문서 생성을 지원합니다. 이미지 삽입은 한컴오피스에서 후처리로 추가하는 것을 권장합니다.

## 10. 주의사항

> [!CAUTION]
> - 모든 생성 후 반드시 **`validate.py`로 검증**이 실행됩니다
> - `.hwp`(바이너리) 파일은 **지원 불가** → `.hwpx`로 다시 저장 필요
> - XML 편집 시 **네임스페이스 접두사** (`hp:`, `hs:`, `hh:`, `hc:`)를 유지해야 합니다
> - `section0.xml`의 첫 문단에는 반드시 **`secPr` + `colPr`** 구조가 포함되어야 합니다
