# 지오코딩 프로세스 사전 테스트 로직 추가 계획

본 계획은 지오코딩 스크립트의 안정성을 확보하기 위해 본격적인 전체 실행 전 샘플 데이터를 통한 API 호출 테스트 단계를 추가하는 것을 목표로 합니다.

## 제안 사항
- **사전 테스트 로직**: 전체 데이터를 처리하기 전, 상위 3개 정도의 샘플 주소를 사용하여 API 호출을 시도합니다.
- **중단 조건**: 테스트 호출이 모두 실패하거나 설정된 기준 이하의 성공률을 보일 경우(예: 3개 중 0개 성공), 스크립트 실행을 즉시 중단하고 에러 메시지를 출력합니다.

## 변경 대상 파일

### 1. geocoding_guideline.md
- **추가 내용**: "4.3 사전 테스트 로직 (Pre-run Validation)" 섹션을 신설하여 표준 코드 스니펫 제공.
- **내용 요약**: 대량 처리 전 API 설정 및 주소 전처리 상태를 점검하는 필수 단계로 정의.

### 2. hospital_geocoding_plan.md
- **추가 내용**: "2.4 지오코딩 실행" 또는 "3. 실행 스크립트" 섹션에 사전 테스트 호출 단계 명시.
- **내용 요약**: 스크립트 요구 사항에 '3개 샘플 테스트 및 실패 시 중단' 기능 추가.
