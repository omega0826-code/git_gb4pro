## 3. 결합 전략

### 3.1 결합 목적

**강남언니 데이터 + HIRA 데이터 결합의 가치**:

1. **플랫폼 등록 병원 특성 분석**
   - 강남언니에 등록된 병원 vs 미등록 병원 비교
   - 등록 병원의 공통 특성 파악

2. **데이터 보완**
   - 강남언니: 평점, 리뷰 등 사용자 평가 데이터
   - HIRA: 공식 의료 인력, 시설, 장비 정보

3. **마케팅 인사이트**
   - 플랫폼 등록 병원의 경쟁력 분석
   - 미등록 병원의 잠재 고객 발굴

---

### 3.2 결합 유형

#### ✅ **권장: Left Join (HIRA 기준)**

```python
# HIRA 데이터를 기준으로 강남언니 데이터 병합
df_hira = pd.read_csv('서울_병원_통합_2025.12.csv')
df_gangnam = pd.read_csv('gangnam_hospitals_final_20260105_065921_update_v1.0.csv')

# 병원명 기반 매칭
df_merged = df_hira.merge(
    df_gangnam,
    left_on='요양기관명',
    right_on='hospital_name',
    how='left',
    indicator=True
)
```

**특징**:
- HIRA 데이터의 모든 병원 유지
- 강남언니 등록 여부를 `_merge` 컬럼으로 확인
- 미등록 병원은 강남언니 컬럼이 NaN

---

#### 대안: Inner Join (등록 병원만)

```python
# 강남언니에 등록된 병원만 분석
df_merged = df_hira.merge(
    df_gangnam,
    left_on='요양기관명',
    right_on='hospital_name',
    how='inner'
)
```

**특징**:
- 강남언니 등록 병원만 포함
- 데이터 크기 축소 (약 1,000행 내외)
- 등록 병원 심층 분석에 적합

---

### 3.3 중복 컬럼 취합 계획

결합 시 HIRA와 강남언니 데이터에서 중복되는 정보가 있습니다. 이러한 중복 컬럼의 처리 방법을 정의합니다.

#### 중복 컬럼 목록

| 정보 유형 | HIRA 컬럼 | 강남언니 컬럼 | 중복 여부 |
|----------|----------|--------------|----------|
| **병원명** | 요양기관명 | hospital_name | ✅ 중복 |
| **주소** | 주소 | address | ✅ 중복 (형식 다름) |
| **지역(시도)** | 시도코드명 | sido | ✅ 중복 |
| **지역(시군구)** | 시군구코드명 | sigungu | ✅ 중복 |
| **의사 정보** | 총의사수 | doctors (리스트) | ⚠️ 부분 중복 |
| **진료과목** | 진료과목_개수 | treatment_tags | ⚠️ 부분 중복 |

---

#### 취합 전략

##### 1. 병원명 (요양기관명 vs hospital_name)

**차이점**:
- HIRA: 공식 등록명 (법인 표기 포함)
- 강남언니: 마케팅용 약칭

**권장 처리**:
```python
# 1. HIRA 병원명을 기본으로 사용
df_merged['병원명'] = df_merged['요양기관명']

# 2. 강남언니 병원명은 별도 컬럼 유지 (비교용)
# hospital_name은 자동으로 hospital_name_강남언니로 suffix 추가됨

# 3. 병원명 차이 플래그
df_merged['병원명_일치'] = (
    df_merged['병원명_정제'] == 
    df_merged['hospital_name'].apply(normalize_hospital_name)
)
```

**결과 컬럼**:
- `요양기관명` (HIRA 공식명) ← **기본 사용**
- `hospital_name` (강남언니 표기) ← 참고용
- `병원명_정제` (매칭용)
- `병원명_일치` (플래그)

---

##### 2. 주소 (주소 vs address)

**차이점**:
- HIRA: 도로명 주소 (상세, 표준화)
- 강남언니: 간략한 주소 (건물명 중심)

**권장 처리**:
```python
# 1. HIRA 주소를 기본으로 사용
df_merged['주소_표준'] = df_merged['주소']

# 2. 강남언니 주소는 참고용
# address는 자동으로 address_강남언니로 suffix 추가됨

# 3. 주소 일치도 검증 (읍면동 기준)
def extract_dong(addr):
    import re
    match = re.search(r'(\w+동|\w+읍|\w+면)', str(addr))
    return match.group(1) if match else ''

df_merged['주소_읍면동_HIRA'] = df_merged['주소'].apply(extract_dong)
df_merged['주소_읍면동_강남언니'] = df_merged['address'].apply(extract_dong)
df_merged['주소_읍면동_일치'] = (
    df_merged['주소_읍면동_HIRA'] == df_merged['주소_읍면동_강남언니']
)
```

**결과 컬럼**:
- `주소` (HIRA 도로명 주소) ← **기본 사용**
- `address` (강남언니 주소) ← 참고용
- `주소_읍면동_일치` (검증용 플래그)

---

##### 3. 지역 정보 (시도/시군구)

**차이점**:
- HIRA: 표준 행정구역명 (예: "서울특별시", "강남구")
- 강남언니: 간략 표기 (예: "서울", "강남")

**권장 처리**:
```python
# 1. HIRA 지역 정보를 기본으로 사용
df_merged['시도'] = df_merged['시도코드명']
df_merged['시군구'] = df_merged['시군구코드명']

# 2. 강남언니 지역 정보는 검증용
# sido, sigungu는 자동으로 sido_강남언니, sigungu_강남언니로 suffix 추가됨

# 3. 지역 일치 검증
df_merged['시도_일치'] = df_merged['시도코드명'].str.contains(
    df_merged['sido'], 
    na=False
)
df_merged['시군구_일치'] = df_merged['시군구코드명'].str.contains(
    df_merged['sigungu'], 
    na=False
)
```

**결과 컬럼**:
- `시도코드명`, `시군구코드명` (HIRA 표준) ← **기본 사용**
- `sido`, `sigungu` (강남언니) ← 검증용
- `시도_일치`, `시군구_일치` (플래그)

---

##### 4. 의사 정보 (총의사수 vs doctors)

**차이점**:
- HIRA: 총의사수 (숫자)
- 강남언니: 의사 이름 리스트 (문자열, 쉼표 구분)

**권장 처리**:
```python
# 1. HIRA 총의사수를 공식 수치로 사용
df_merged['의사수_공식'] = df_merged['총의사수']

# 2. 강남언니 의사 수 계산
df_merged['의사수_강남언니'] = df_merged['doctors'].str.split(',').str.len()

# 3. 의사 수 차이 분석
df_merged['의사수_차이'] = (
    df_merged['의사수_공식'] - df_merged['의사수_강남언니']
)

# 4. 강남언니 의사 리스트는 별도 유지 (상세 정보)
# doctors 컬럼 유지
```

**결과 컬럼**:
- `총의사수` (HIRA 공식) ← **기본 사용**
- `의사수_강남언니` (강남언니 등록 의사 수)
- `doctors` (강남언니 의사 리스트) ← 상세 정보
- `의사수_차이` (분석용)

---

##### 5. 진료과목 정보 (진료과목_개수 vs treatment_tags)

**차이점**:
- HIRA: 진료과목 개수 (표준 진료과목 코드 기준)
- 강남언니: 진료 태그 (마케팅용, 시술 중심)

**권장 처리**:
```python
# 1. HIRA 진료과목 개수를 공식 수치로 사용
df_merged['진료과목수_공식'] = df_merged['진료과목_개수']

# 2. 강남언니 진료 태그 개수 계산
df_merged['진료태그수'] = df_merged['treatment_tags'].str.split(',').str.len()

# 3. 두 정보는 별도 유지 (분류 체계가 다름)
# - 진료과목_개수: 공식 진료과목 (내과, 외과 등)
# - treatment_tags: 시술/치료 태그 (보톡스, 필러 등)
```

**결과 컬럼**:
- `진료과목_개수` (HIRA 공식) ← **기본 사용**
- `treatment_tags` (강남언니 태그) ← 마케팅 정보
- `진료태그수` (강남언니 태그 개수)

---

#### 중복 컬럼 처리 원칙

| 원칙 | 설명 | 적용 |
|------|------|------|
| **1. HIRA 우선** | 공식 데이터는 HIRA 기준 사용 | 병원명, 주소, 지역, 의사수 |
| **2. 강남언니 보완** | 강남언니는 추가 정보로 활용 | 평점, 리뷰, 진료태그 |
| **3. 검증 플래그** | 중복 정보는 일치 여부 검증 | 주소_일치, 시군구_일치 |
| **4. Suffix 활용** | 강남언니 컬럼은 `_강남언니` suffix | address_강남언니 등 |
| **5. 별도 유지** | 분류 체계가 다른 경우 모두 유지 | 진료과목 vs 진료태그 |

### 4.1 병원명 기반 매칭의 문제점

**문제 1: 병원명 표기 차이**

| HIRA | 강남언니 | 매칭 여부 |

| 더피부과의원 | 더피부과 | ❌ 불일치 |

- 주소 정보로 추가 검증 필요

### 4.2 병원명 정제 (Normalization)

#### STEP 1: 병원명 정제 함수

```python
import re

def normalize_hospital_name(name):
    """병원명 정제 함수"""
    if pd.isna(name):
        return ""
    
    # 1. 소문자 변환
    name = str(name).strip()
    
    # 2. 법인 표기 제거
    name = re.sub(r'\(주\)|\(의\)|\(재\)|\(사\)', '', name)
    
    # 3. 공백 제거
    name = re.sub(r'\s+', '', name)
    
    # 4. 특수문자 제거
    name = re.sub(r'[^\w가-힣]', '', name)
    
    # 5. 의원/병원 통일
    name = name.replace('의원', '').replace('병원', '')
    
    return name

# 적용
df_hira['병원명_정제'] = df_hira['요양기관명'].apply(normalize_hospital_name)
df_gangnam['병원명_정제'] = df_gangnam['hospital_name'].apply(normalize_hospital_name)
```

#### STEP 2: Fuzzy Matching (유사도 기반 매칭)

```python
from fuzzywuzzy import fuzz

def fuzzy_match_hospitals(hira_name, gangnam_names, threshold=85):
    """
    퍼지 매칭으로 가장 유사한 병원명 찾기
    
    Parameters:
    - hira_name: HIRA 병원명
    - gangnam_names: 강남언니 병원명 리스트
    - threshold: 유사도 임계값 (0-100)
    
    Returns:
    - 매칭된 병원명, 유사도 점수
    """
    best_match = None
    best_score = 0
    
    for gn_name in gangnam_names:
        score = fuzz.ratio(hira_name, gn_name)
        if score > best_score and score >= threshold:
            best_score = score
            best_match = gn_name
    
    return best_match, best_score

# 예시
gangnam_names = df_gangnam['병원명_정제'].tolist()
hira_name = "더피부과"
match, score = fuzzy_match_hospitals(hira_name, gangnam_names, threshold=85)
print(f"매칭: {match}, 유사도: {score}")
```

#### STEP 3: 주소 기반 검증

```python
def verify_by_address(hira_addr, gangnam_addr):
    """주소 기반 매칭 검증"""
    # 읍면동 추출
    hira_dong = extract_dong(hira_addr)
    gangnam_dong = extract_dong(gangnam_addr)
    
    # 동 일치 여부
    return hira_dong == gangnam_dong

def extract_dong(address):
    """주소에서 읍면동 추출"""
    import re
    match = re.search(r'(\w+동|\w+읍|\w+면)', address)
    return match.group(1) if match else ""
```

### 4.3 매칭 알고리즘 (3단계)

```python
def match_hospitals(df_hira, df_gangnam):
    """
    3단계 매칭 알고리즘
    1. 정제된 병원명 완전 일치
    2. 퍼지 매칭 (유사도 85% 이상)
    3. 주소 검증
    """
    results = []
    
    for idx, hira_row in df_hira.iterrows():
        hira_name_norm = hira_row['병원명_정제']
        hira_addr = hira_row['주소']
        hira_dong = hira_row['읍면동']
        
        # 1단계: 완전 일치
        exact_match = df_gangnam[
            df_gangnam['병원명_정제'] == hira_name_norm
        ]
        
        if len(exact_match) > 0:
            # 주소 검증
            for _, gn_row in exact_match.iterrows():
                if verify_by_address(hira_addr, gn_row['address']):
                    results.append({
                        'hira_idx': idx,
                        'gangnam_id': gn_row['id'],
                        'match_type': 'exact',
                        'confidence': 100
                    })
                    break
        else:
            # 2단계: 퍼지 매칭
            gangnam_names = df_gangnam['병원명_정제'].tolist()
            match, score = fuzzy_match_hospitals(
                hira_name_norm, 
                gangnam_names, 
                threshold=85
            )
            
            if match:
                gn_row = df_gangnam[
                    df_gangnam['병원명_정제'] == match
                ].iloc[0]
                
                # 3단계: 주소 검증
                if verify_by_address(hira_addr, gn_row['address']):
                    results.append({
                        'hira_idx': idx,
                        'gangnam_id': gn_row['id'],
                        'match_type': 'fuzzy',
                        'confidence': score
                    })
    
    return pd.DataFrame(results)
```

## 7. 결측치 처리 방법 검토

### 7.1 결측치 유형 분류

결합 후 발생하는 결측치는 크게 3가지 유형으로 분류됩니다.

#### 유형 1: 구조적 결측치 (Structural Missing)

**정의**: 데이터 특성상 당연히 존재하지 않는 값

**예시**:
- 미등록 병원의 강남언니 컬럼 (rating, review_count 등)
- 의원급의 병상 정보
- 특정 진료과목이 없는 병원의 해당 전문의 수

**권장 처리 방법**:
- 결측치 유지 또는 0으로 표시
- 별도 플래그 컬럼 생성 (예: `평점_있음`)

---

#### 유형 2: 데이터 누락 (Data Missing)

**정의**: 원본 데이터에서 수집되지 않은 값

**예시**:
- HIRA 데이터의 병원홈페이지 (83.9% 누락)
- 응급실 정보 (73.5% 누락)
- 강남언니의 일부 병원 소개글

**권장 처리 방법**:
- **50% 이상 누락**: 분석에서 제외 또는 별도 처리
- **10-50% 누락**: 대체값 또는 플래그 컬럼
- **10% 미만**: 대체값 또는 행 제거

---

#### 유형 3: 매칭 실패 (Matching Failure)

**정의**: 매칭 과정에서 발생한 결측치

**예시**:
- 병원명 불일치로 매칭 실패
- 주소 검증 실패

**권장 처리 방법**:
- 매칭 실패 원인 분석
- 수동 매칭 검토
- 신뢰도 낮은 경우 제외

---

### 7.2 결측치 처리 전략

#### 전략 1: 유지 (Keep as Missing)

**적용 대상**: 구조적 결측치, 분석에 영향 없는 컬럼

**장점**: 데이터 왜곡 없음  
**단점**: 일부 분석 함수에서 오류 발생 가능

---

#### 전략 2: 0으로 대체

**적용 대상**: 수치형 데이터 중 "없음"을 의미하는 경우

**예시**: `review_count`, `event_count`, `장비_CT_대수`

**장점**: 집계 분석 용이  
**단점**: 0과 결측치 구분 불가

---

#### 전략 3: 기본값으로 대체

**적용 대상**: 범주형 데이터, 텍스트 데이터

**예시**: 
- `병원홈페이지` → '정보없음'
- `introduction` → '소개글 없음'
- `treatment_tags` → '태그 없음'

**장점**: 명시적 표현  
**단점**: 분석 시 별도 처리 필요

---

#### 전략 4: 플래그 컬럼 생성

**적용 대상**: 결측 여부가 중요한 정보인 경우

**예시**:
- `홈페이지_있음` = `병원홈페이지`.notna()
- `평점_있음` = `rating`.notna()
- `응급실_있음` = `응급실_주간_운영여부`.notna()

**장점**: 결측 패턴 분석 가능  
**단점**: 컬럼 수 증가

---

#### 전략 5: 조건부 대체

**적용 대상**: 다른 컬럼 값에 따라 결측치 의미가 다른 경우

**예시**:
- 의원급은 병상 정보가 없는 것이 정상 → 0으로 대체
- 등록 병원인데 평점 없음 → 데이터 품질 이슈

**장점**: 맥락 고려한 처리  
**단점**: 복잡한 로직 필요

---

### 7.3 결측치 처리 체크리스트

#### 처리 전 확인사항

- [ ] 결측치 비율 확인 (전체, 컬럼별)
- [ ] 결측치 패턴 분석 (종별, 지역별)
- [ ] 결측치 유형 분류 (구조적/누락/매칭실패)
- [ ] 분석 목적에 따른 처리 전략 수립

#### 처리 후 검증사항

- [ ] 결측치 처리 로그 기록
- [ ] 처리 전후 통계 비교
- [ ] 이상값 발생 여부 확인
- [ ] 분석 결과 타당성 검증

### 7.4 권장 처리 방침

```python
def validate_matching_quality(df_merged):
    """매칭 품질 검증"""
    
    print("=== 매칭 품질 검증 ===\n")
    
    # 1. 매칭률
    total = len(df_merged)
    matched = df_merged['강남언니_등록'].sum()
    match_rate = matched / total * 100
    
    print(f"1. 매칭률: {match_rate:.1f}% ({matched:,}/{total:,})")
    
    if match_rate < 50:
        print("   ⚠️ 경고: 매칭률이 50% 미만입니다.")
        print("   → 병원명 정제 확인 필요")

    # 2. 지역 일치도
    if '시군구_일치' in df_merged.columns:
        region_match = df_merged[df_merged['강남언니_등록'] == True]['시군구_일치'].mean() * 100

            print("   → 동명이병원 가능성 확인")
        else:
            print("   ✅ 정상")
    
    # 3. 중복 매칭 확인
    if 'id' in df_merged.columns:
        duplicates = df_merged['id'].duplicated().sum()
        print(f"\n3. 중복 매칭: {duplicates}건")
        
        if duplicates > 0:
            print("   ⚠️ 경고: 중복 매칭이 발견되었습니다.")

        else:
            print("   ✅ 중복 없음")
    
    return {
        'match_rate': match_rate,
        'region_match': region_match if '시군구_일치' in df_merged.columns else None,
        'duplicates': duplicates if 'id' in df_merged.columns else 0
    }
```

```python
def validate_data_consistency(df_merged):
    """데이터 일관성 검증"""
    
    print("=== 데이터 일관성 검증 ===\n")
    
    errors = []
    
    # 1. 행 수 검증
    expected_rows = len(df_hira)  # 원본 HIRA 데이터 행 수
    actual_rows = len(df_merged)
    
    if expected_rows != actual_rows:
        errors.append(f"행 수 불일치: 예상 {expected_rows:,}, 실제 {actual_rows:,}")
    else:
        print(f"1. 행 수: ✅ {actual_rows:,}행")
    
    # 2. 필수 컬럼 존재 확인
    required_cols = ['암호화요양기호', '요양기관명', '시군구코드명', '강남언니_등록']
    missing_cols = [col for col in required_cols if col not in df_merged.columns]
    
    if missing_cols:
        errors.append(f"필수 컬럼 누락: {missing_cols}")
    else:
        print(f"2. 필수 컬럼: ✅ 모두 존재")
    
    # 3. 중복 행 확인
    duplicates = df_merged['암호화요양기호'].duplicated().sum()
    
    if duplicates > 0:
        errors.append(f"중복 행 발견: {duplicates}건")
    else:
        print(f"3. 중복 행: ✅ 없음")
    
    # 4. 데이터 타입 확인
    numeric_cols = ['총의사수', 'rating', 'review_count']
    for col in numeric_cols:
        if col in df_merged.columns:
            if not pd.api.types.is_numeric_dtype(df_merged[col]):
                errors.append(f"데이터 타입 오류: {col}이(가) 숫자형이 아님")
    
    if not errors:
        print(f"4. 데이터 타입: ✅ 정상")
    
    # 5. 결과 출력
    if errors:
        print("\n⚠️ 검증 실패:")
        for error in errors:
            print(f"   - {error}")
        return False
    else:
        print("\n✅ 모든 검증 통과")
        return True

# 실행
```

```python
import pandas as pd
from fuzzywuzzy import fuzz
import re

# 1. 데이터 로드
df_hira = pd.read_csv('서울_병원_통합_2025.12.csv', encoding='utf-8-sig')
df_gangnam = pd.read_csv(
    'gangnam_hospitals_final_20260105_065921_update_v1.0.csv',
    encoding='utf-8-sig'
)

# 2. 병원명 정제
def normalize_hospital_name(name):
    if pd.isna(name):
        return ""
    name = str(name).strip()
    name = re.sub(r'\(주\)|\(의\)|\(재\)|\(사\)', '', name)
    name = re.sub(r'\s+', '', name)
    name = re.sub(r'[^\w가-힣]', '', name)
    return name

df_hira['병원명_정제'] = df_hira['요양기관명'].apply(normalize_hospital_name)
df_gangnam['병원명_정제'] = df_gangnam['hospital_name'].apply(normalize_hospital_name)

# 3. 매칭 (간단 버전 - 완전 일치만)
df_merged = df_hira.merge(
    df_gangnam,
    left_on='병원명_정제',
    right_on='병원명_정제',
    how='left',
    suffixes=('', '_강남언니')
)

# 4. 강남언니 등록 플래그
df_merged['강남언니_등록'] = df_merged['id'].notna()

# 5. 저장
output_path = 'HIRA_강남언니_결합_2025.12.csv'
df_merged.to_csv(output_path, index=False, encoding='utf-8-sig')

print(f"결합 완료: {output_path}")
print(f"총 병원: {len(df_merged):,}")
print(f"강남언니 등록: {df_merged['강남언니_등록'].sum():,}")
```

```python
def generate_merge_statistics_report(df_merged, output_path):
    """결합 통계 리포트 생성"""
    
    from datetime import datetime
    
    report = []
    report.append("# HIRA + 강남언니 데이터 결합 통계 리포트\n")
    report.append(f"> **작성일**: {datetime.now().strftime('%Y-%m-%d')}  ")
    report.append("> **데이터 기준일**: 2025년 12월  \n")
    
    # 1. 결합 개요
    total = len(df_merged)
    matched = df_merged['강남언니_등록'].sum()
    not_matched = total - matched
    
    report.append("## 1. 결합 개요\n")
    report.append(f"- **HIRA 총 병원 수**: {total:,}개")
    report.append(f"- **강남언니 등록 병원**: {matched:,}개 ({matched/total*100:.1f}%)")
    report.append(f"- **미등록 병원**: {not_matched:,}개 ({not_matched/total*100:.1f}%)\n")
    
    # 2. 매칭 결과
    if '매칭_유형' in df_merged.columns:
        report.append("## 2. 매칭 결과\n")
        report.append("### 2.1 매칭 유형별 분포")
        match_types = df_merged['매칭_유형'].value_counts()
        for match_type, count in match_types.items():
            report.append(f"- {match_type}: {count:,}개 ({count/matched*100:.1f}%)")
    
    # 파일 저장
    with open(output_path, 'w', encoding='utf-8') as f:
        f.write('\n'.join(report))
    
    print(f"✅ 통계 리포트 생성: {output_path}")

# 실행
timestamp = datetime.now().strftime('%Y%m%d')
report_path = f'reports/결합_통계_리포트_{timestamp}.md'
generate_merge_statistics_report(df_merged, report_path)
```
